# langgraph-triage-demo-ui-answer

Browser front end for the LangGraph triage agent built in the [demo on goldenpassport.blog](https://goldenpassport.blog/blog/automation-review-langgraph-demo). Step 9's companion project.

Upload a `.docx`, watch the graph route it, see the report. Real model calls via local Ollama.

## What it does

- **Single-page React app** built with Vite + TypeScript
- **.docx upload** parsed in-browser with [mammoth](https://github.com/mwilliamson/mammoth.js)
- **Live workflow visualisation** — each graph node lights up as it runs (idle → running → done), with skipped branches greyed out
- **Real Ollama model calls** for `classify` and `extract` (uses the locally-running [llama3.2:3b](https://ollama.com/library/llama3.2) by default)
- **Deterministic routing** — `validate` and `decide` are pure functions, so the audit trail is unambiguous
- **Three sample .docx files** generated by script, one per branch outcome

## Prerequisites

| Tool | Version | Why |
| --- | --- | --- |
| **Node.js** | 20+ | Vite 5 + LangGraph 1.x |
| **pnpm** | 9+ | (npm or yarn also fine; commands below assume pnpm) |
| **Ollama** | latest | Hosts the local model. Install from [ollama.com](https://ollama.com) |

## Setup

### 1. Install Ollama and pull the model

```bash
# macOS (homebrew)
brew install ollama

# Linux
curl -fsSL https://ollama.com/install.sh | sh
```

Start the Ollama server (if not already running as a background service):

```bash
ollama serve
```

Pull the model:

```bash
ollama pull llama3.2:3b
```

Verify it works:

```bash
ollama run llama3.2:3b "Say hello"
```

You should see a model response.

### 2. Install + run this project

```bash
pnpm install
pnpm generate-samples   # creates 3 .docx files in samples/
pnpm dev                # starts Vite on http://localhost:5173
```

Open http://localhost:5173 in your browser, click **Choose .docx file**, and upload one of `samples/clean-claim.docx`, `samples/malformed-claim.docx`, or `samples/unknown.docx`.

## How it talks to Ollama

The Vite dev server proxies `/api/ollama/*` to `http://localhost:11434/*` (Ollama's default port). This avoids CORS in development. In production you would either:

- Run a real backend that proxies to Ollama (or to a hosted Ollama-compatible endpoint)
- Set `OLLAMA_ORIGINS=*` on the Ollama server (less secure; loosens its CORS allow-list)

The proxy config is in `vite.config.ts`.

## Expected behaviour

| Upload | Category | Decision | Branch |
| --- | --- | --- | --- |
| `clean-claim.docx` | claim | **auto-process** | Green tick on the right path |
| `malformed-claim.docx` | claim | **human-review** | Amber on the centre path (missing policy + date) |
| `unknown.docx` | unknown | **reject** | Red on the left path |

Each upload takes 3–8 seconds depending on your machine's CPU/GPU. The graph view animates as each node runs.

## Editing the samples

The samples are generated from the strings in [`samples/generate-samples.ts`](./samples/generate-samples.ts). Edit any of the paragraph arrays, then re-run:

```bash
pnpm generate-samples
```

Refresh the browser and re-upload.

## Where to take this next

- **Swap the model**. Change `MODEL_NAME` in [`src/lib/nodes.ts`](./src/lib/nodes.ts) to any other Ollama-installed model (e.g. `qwen2.5:3b`, `phi3:3.8b`, `mistral:7b`). Pull it first with `ollama pull <model>`.
- **Use structured-output instead of text parsing**. LangGraph supports `model.withStructuredOutput(schema)` for JSON-schema-driven extraction. Small models are unreliable with that, but 7B+ models handle it well.
- **Add a checkpointer**. The browser-side LangGraph supports in-memory checkpointing. Add `{ checkpointer: new MemorySaver() }` to `.compile()` and you can pause / resume the graph (e.g. for a real human-review approval UI).
- **Replace Ollama with a hosted model**. Drop `@langchain/ollama` for `@langchain/openai` or `@langchain/anthropic`, point at their endpoint, and supply the API key via an env var. The graph stays identical.
- **Add a Mermaid diagram view**. `graph.getGraph().drawMermaid()` returns a string you can render alongside the SVG visualisation.
